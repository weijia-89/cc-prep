Q4, baked-aside markdown syntax

What does Wei type in CC_Study_Guide.md to mark a permanent annotation?

Option A, raw HTML

Markdown source
<aside class="user-note" data-id="note-rpo">
RPO vs RTO is the #1 trap. Drill these
together every Anki session.
</aside>
Rendered output
RPO vs RTO is the #1 trap. Drill these together every Anki session.

Option B, attrs_plugin pattern

Markdown source
*RPO vs RTO is the #1 trap. Drill these
together every Anki session.*{.user-note #note-rpo}
Rendered output
RPO vs RTO is the #1 trap. Drill these together every Anki session.

Option C, custom blockquote prefix

Markdown source
> [note] RPO vs RTO is the #1 trap.
> Drill these together every Anki session.
Rendered output
RPO vs RTO is the #1 trap. Drill these together every Anki session.

Tradeoff: Option A is most explicit, no plugin work needed (html: True already set), but verbose. Option B uses the attrs_plugin already loaded; one line in markdown, but the {.user-note} attribute syntax is unfamiliar if you forget it. Option C needs a small custom transform in render_markdown() (~20 LOC) to convert > [note] ... blockquotes into <aside class="user-note">...</aside>. Most natural to type, most code to add.

Q5, localStorage scoping under file://

Chrome treats every file://-loaded page as a separate origin. That means notes stored in day-01.html's localStorage are invisible from day-02.html or guide.html, even though they share a directory. Three handlings:

OptionBehaviorCost
A: accept per-page-originEach HTML file has its own notes store. Notes anchored to guide.html#23-recovery-objectives only visible when viewing guide.html (which is fine; that's the page they live on).Zero; just document the behavior.
B: require local http.serverAll pages share http://localhost:8765 origin. Notes addressable across pages by URL. Wei runs python3 -m http.server 8765 from study_guide/daily/ once per study session.One terminal window during cram. Wei has to remember to start the server.
C: hybrid (auto-detect)Page loads, JS checks window.location.protocol. If file://, show a one-line banner at top: "Notes are per-page. Run python3 -m http.server 8765 for cross-page." If http://, no banner.~10 LOC banner code.

For a 13-day personal cram tool, A is probably enough. Notes anchored to a heading live on that page; you only need cross-page if you take notes on day-01.html and want them visible from guide.html, which is rare.
